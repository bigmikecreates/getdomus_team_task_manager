# Manager Workflow

As a manager, you can create tasks, assign developers, and track team progress.

## Creating a Task

<!-- SCREENSHOT: create-task-form
     What to capture: Task creation form with all fields visible
     Suggested size: 1280x800
-->

1. Navigate to **Tasks** from the sidebar
2. Click **+ New Task**
3. Fill in the details:
   - **Title** (required) — What needs to be done
   - **Description** (optional) — Additional context
   - **Status** — Starting status (default: To Do)
   - **Priority** — Low, Medium, High, or Critical
   - **Due Date** (optional) — When it should be completed
4. Click **Create Task**

You'll be redirected to the task list where your new task appears.

## Assigning Developers

<!-- SCREENSHOT: assign-developers
     What to capture: Task edit page showing developer assignment dropdown
     Suggested size: 1280x800
-->

1. Open a task from the task list
2. Click **Edit**
3. Use the developer dropdown to select team members
4. You can assign multiple developers to a single task
5. Click **Save**

## Updating a Task

1. Open the task from the task list
2. Click **Edit**
3. Change any fields (status, priority, description, etc.)
4. Click **Save**

## Viewing Task Details

<!-- SCREENSHOT: task-detail-view
     What to capture: Task detail page showing properties, assigned developers, action buttons
     Suggested size: 1280x800
-->

Click any task title in the list to open the detail view. From here you can:

- See all task properties at a glance
- View which developers are assigned
- Edit or delete the task

## Deleting a Task

1. Open the task detail view
2. Click **Delete**
3. Confirm the deletion in the dialog

!!! warning
    Deleting a task is permanent. All developer assignments will also be removed.

## Tracking Progress

<!-- SCREENSHOT: dashboard-stats
     What to capture: Dashboard showing task statistics and attention required section
     Suggested size: 1280x800
-->

Use the **Dashboard** to monitor:

- **Task statistics** — Total tasks broken down by status and priority
- **Attention Required** — Overdue and critical tasks that need immediate focus
- **Upcoming tasks** — Tasks due soon

Click any task in the Attention Required section to go directly to it.

## Sorting the Task List

<!-- SCREENSHOT: sortable-columns
     What to capture: Task list with sort indicator visible on a column
     Suggested size: 1280x800
-->

Click any column header to sort the task list:

- **Status** — Sort by workflow stage
- **Priority** — Sort by urgency
- **Due Date** — Sort by deadline
- **Assigned** — Sort by number of assigned developers

Click again to reverse the sort order.
