# Developer Workflow

As a developer, you can view your assigned tasks and update their status.

## Viewing Your Tasks

<!-- SCREENSHOT: task-list-developer
     What to capture: Task list showing only tasks assigned to the logged-in developer
     Suggested size: 1280x800
-->

1. Navigate to **Tasks** from the sidebar
2. The list shows all tasks in the project
3. Tasks assigned to you are highlighted

## Updating Task Status

1. Open a task from the task list
2. Click **Edit**
3. Change the **Status** field:
   - **To Do** — Not started yet
   - **In Progress** — Actively working on it
   - **In Review** — Ready for review
   - **Done** — Completed
4. Click **Save**

## Viewing Task Details

Click any task title to see:

- Full description
- Priority and due date
- Which other developers are assigned
- Current status

<!-- SCREENSHOT: task-detail-developer

-->

!!! note
    Developers cannot create, delete, or reassign tasks. These actions are restricted to managers and admins.

## Viewing Team Availability

<!-- SCREENSHOT: developers-availability
     What to capture: Developers page showing local time and availability columns
     Suggested size: 1280x800
-->

Navigate to **Developers** from the sidebar to see:

- Team member names and timezones
- **Local Time** — Each developer's current local time
- **Availability** — Whether they're within their working hours
- **Online Status** — Green dot indicates recent activity

This is useful for knowing when to reach out to teammates in different time zones.
